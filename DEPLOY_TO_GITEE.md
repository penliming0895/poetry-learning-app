# 部署到 Gitee Pages - 操作步骤

## 🎯 目标
将古诗词背默应用部署到 Gitee Pages，国内学生可以直接访问，无需 VPN。

## 📋 已完成的准备工作

✅ 已配置 Next.js 静态导出
✅ 已删除需要后端 API 的功能（TTS 语音）
✅ 已测试构建成功
✅ 代码已推送到 GitHub

## 🚀 您需要完成的步骤

### 步骤 1：注册 Gitee 账号

1. 访问 https://gitee.com
2. 点击右上角"注册"
3. 填写信息完成注册（如果有账号则登录）

---

### 步骤 2：创建 Gitee 仓库

1. 登录后，点击右上角 **"+"** → **新建仓库**
2. 填写以下信息：
   - **仓库名称**：`poetry-learning-app`（推荐使用这个名称）
   - **仓库介绍**：`初三语文古诗词背默应用`
   - **是否公开**：选择 **公开**
   - **初始化仓库**：全部**不勾选**
3. 点击 **创建**

**记住你的仓库地址，格式类似：**
```
https://gitee.com/你的用户名/poetry-learning-app.git
```

---

### 步骤 3：推送代码到 Gitee

打开终端（命令行），在项目目录下运行以下命令：

```bash
# 1. 添加 Gitee 远程仓库（替换为你的仓库地址）
git remote add gitee https://gitee.com/你的用户名/poetry-learning-app.git

# 2. 推送代码到 Gitee
git push gitee main
```

**示例（假设你的用户名是 zhangsan）：**
```bash
git remote add gitee https://gitee.com/zhangsan/poetry-learning-app.git
git push gitee main
```

---

### 步骤 4：开启 Gitee Pages 服务

1. 在 Gitee 仓库页面
2. 点击顶部菜单的 **服务** → **Gitee Pages**
3. 点击 **启用 Gitee Pages 服务**
4. 填写配置：
   - **构建分支**：选择 `main`
   - **构建目录**：填写 `out`
   - **部署分支**：选择 `main`
   - **目录**：填写 `out`
5. 点击 **启动**

**注意**：首次启动可能需要 1-3 分钟。

---

### 步骤 5：获取访问地址

部署成功后，你会看到：

```
网站地址：https://你的用户名.gitee.io/poetry-learning-app
```

**这就是分享给学生的链接！**

---

## ✅ 验证部署

打开浏览器，访问你的 Gitee Pages 地址。

**如果能看到首页，说明部署成功！** 🎉

## 📝 日常更新部署

当你修改代码后，更新部署：

```bash
# 1. 构建
pnpm build

# 2. 提交
git add -A
git commit -m "描述你的修改"
git push gitee main

# 3. 在 Gitee Pages 页面点击"更新"按钮（在右上角）
```

## ⚠️ 重要提示

### 功能说明

由于是静态部署，以下功能**不可用**：
- ❌ 语音朗读功能（TTS）

其他功能**正常可用**：
- ✅ 每日背默5首
- ✅ 背诵练习模式
- ✅ 默写测试模式
- ✅ 错题本
- ✅ 成就系统
- ✅ 进度自动保存

### 为什么需要删除语音功能？

静态网站（Gitee Pages）不支持后端 API，语音朗读需要后端服务支持。

如果未来需要语音功能，可以考虑：
1. 使用支持后端服务的国内平台（如阿里云、腾讯云）
2. 或者使用浏览器自带的语音合成（Web Speech API）

## 🆘 遇到问题？

如果遇到问题，请检查：

1. **Gitee Pages 启动失败**
   - 确保构建目录填写的是 `out`
   - 确保代码已成功推送到 Gitee
   - 查看 Gitee Pages 的错误日志

2. **访问显示 404**
   - 等待 2-3 分钟，让部署完成
   - 检查 URL 是否正确
   - 确认 Gitee Pages 服务已启动

3. **页面样式错乱**
   - 清除浏览器缓存
   - 按 `Ctrl + Shift + R` 强制刷新
